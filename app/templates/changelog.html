Historial de actualizaciones y mejoras en el sistema

Versión 2.1.0 (Monitoreo Avanzado, OOF y Alertas)

Se han implementado con éxito tres nuevas capacidades al panel de administración:

1. Reporte de Licencias Ociosas
  • Optimización de Costos: Escaneo automatizado de sign-ins en Entra ID de los últimos 30 días para usuarios con licencias de Microsoft 365 asignadas.
  • Interfaz de Reporte: Listado en caliente de cuentas inactivas o cuentas que nunca han iniciado sesión, permitiendo desasignar licencias subutilizadas.
2. Configuración de Fuera de Oficina (OOF)
  • Respuestas Automáticas: Modificación en tiempo real del estado de auto-respuestas (Desactivado, Siempre Activo, Programado por fecha/hora).
  • Público Externo e Interno: Formularios dedicados para configurar mensajes diferenciados para remitentes internos y públicos externos (cualquier remitente o contactos únicamente).
3. Alertas de Seguridad en Tiempo Real
  • Detección de Nuevos Administradores: Auditoría proactiva en cada ciclo de sincronización de nuevos administradores globales en los tenants de los clientes.
  • Alertas Inmediatas: Envío automático de notificaciones a través de los canales configurados (Telegram y correo electrónico) ante cualquier asignación detectada.

Versión 2.0.0 (Fase 2 de Implementación)

Se han implementado con éxito tres capacidades principales al panel de administración:

1. Panel de Auditoría Visual (Audit Logs)
  • Bitácora del Sistema: Registro persistente en base de datos de cada acción ejecutada (desbloqueo de cuentas, reseteo de claves, offboarding, gestión de delegados).
  • Filtros Avanzados: Panel interactivo que permite filtrar en tiempo real por Organización, Acción, Estado (Éxito/Fallo) y búsqueda por correo/UPN del operador o destinatario.
2. Gestión de Buzones Compartidos (Shared Mailboxes)
  • Administración Completa: Visualización en tiempo real de todos los buzones de tipo compartido en el tenant.
  • Creación de Buzones: Formulario directo para crear nuevos buzones de correo compartidos indicando nombre, alias y dominio del tenant.
  • Gestión de Delegados: Modificación instantánea de permisos de Read and Manage (Editor) y Send As (Autor) en caliente (agregar y quitar delegados).
3. Motor de Cumplimiento de Seguridad (Compliance Engine)
  • Evaluación de Entra ID: Escaneo automatizado de políticas clave de protección del tenant.
  • Indicadores Clave (Score Cards):
    • MFA Global (Security Defaults)
    • Políticas de MFA vía Conditional Access
    • Bloqueo de Autenticación Heredada (Legacy Auth)
    • Configuración de Expiración de Contraseñas (Password Policies)
  • Recomendaciones en Caliente: Alertas dinámicas con acciones sugeridas para corregir vulnerabilidades del tenant.

Script de Automatización de Onboarding (PowerShell)

Para registrar rápidamente un cliente (Tenant) en la plataforma con los permisos requeridos en Azure AD / Entra ID, puedes ejecutar este script en PowerShell como administrador. Automatiza el registro, asignación de roles de API de Microsoft Graph y genera el secreto de cliente.

Permisos Otorgados por el Script (Tipo Aplicación):
Permiso Propósito
User.ReadWrite.All Bloqueo/desbloqueo de cuentas, reseteo de contraseñas y lecturas de perfil.
Directory.ReadWrite.All Administración de miembros en grupos de seguridad.
Organization.Read.All Lectura de políticas de expiración de contraseñas organizacionales.
MailboxSettings.ReadWrite Creación de buzones compartidos, delegación de accesos y configuración de respuestas automáticas (Fuera de Oficina / OOF).
Policy.Read.All Auditoría de políticas de conditional access e identity security defaults.
Código del Script (PowerShell):
# 1. Conectar al Tenant del Cliente
Connect-MgGraph -Scopes "Application.ReadWrite.All", "AppRoleAssignment.ReadWrite.All"

$AppName = "CloudManager-EnterpriseLicensing"
$RequiredPermissions = @(
    @{ Name = "User.ReadWrite.All"; Id = "741f0c66-ca9a-4c26-850d-8d386c73af46" },
    @{ Name = "Directory.ReadWrite.All"; Id = "19ec0e23-833e-43d4-9197-0c11ceb699c9" },
    @{ Name = "Organization.Read.All"; Id = "296ce679-fb92-411a-8e54-3e970105bfe3" },
    @{ Name = "MailboxSettings.ReadWrite"; Id = "6931bccd-447a-43d1-b442-00a195474933" },
    @{ Name = "Policy.Read.All"; Id = "246ad9d1-6b70-464a-ad1b-324c4355523a" }
)

# 2. Crear App Registration
Write-Host "Creando App Registration..."
$App = New-MgApplication -DisplayName $AppName -SignInAudience "AzureADMyOrg"
$Sp = New-MgServicePrincipal -AppId $App.AppId

# 3. Asignar Permisos
$GraphSp = Get-MgServicePrincipal -Filter "AppId eq '00000003-0000-0000-c000-000000000000'"
foreach ($Perm in $RequiredPermissions) {
    $AppRole = $GraphSp.AppRoles | Where-Object { $_.Id -eq $Perm.Id }
    New-MgServicePrincipalAppRoleAssignment -ServicePrincipalId $Sp.Id -BodyParameter @{
        PrincipalId = $Sp.Id
        ResourceId = $GraphSp.Id
        AppRoleId = $AppRole.Id
    }
}

# 4. Generar Secreto
$Secret = Add-MgApplicationPassword -ApplicationId $App.Id -PasswordCredential @{
    DisplayName = "CloudManager Secret"
    EndDateTime = (Get-Date).AddYears(2)
}

# 5. Mostrar credenciales finales
Clear-Host
Write-Host "REGISTRO COMPLETADO"
Write-Host "Tenant ID: " (Get-MgContext).TenantId
Write-Host "Client ID: " $App.AppId
Write-Host "Client Secret: " $Secret.SecretText
Disconnect-MgGraph
